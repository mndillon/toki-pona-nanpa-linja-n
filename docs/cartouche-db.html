How to use this page

This database controls how tokiponized proper names are displayed as cartouches across all renderer pages. Each entry is looked up automatically when the renderer encounters a capitalised word sequence, e.g. Towasi.

Table columns

  • Name (words) — The tokiponized proper name. Amber words are nanpa-linja-n numeric sequences — they are detected automatically and are always rendered as one cartouche.
  • @db — Only shown for entries containing a nanpa-linja-n numeric sequence. When checked, the entry supports the @db suffix (see below) to override automatic nanpa-linja-n numeric rendering with a hand-picked glyph cartouche.
  • Merge — When checked, all non-numeric words in the entry are merged into a single cartouche. When unchecked, each word gets its own cartouche.
  • Mode — random: each letter maps to a random toki pona word starting with that letter (chosen once per page load, consistent across the page). preferred: use the cartouche you set via the Edit button. ignore: suppress this entry entirely — the words pass through to the renderer as-is.
  • Cartouche (latin desc) — Shows what will be sent to the renderer. → random means it will be generated at page load. (@db) marks the part that is only used when the @db suffix is present in the input.

Editing a cartouche

Click ✏ edit to open the slot editor. Each letter of the name gets a slot — pick a toki pona word starting with that letter from the glyph picker, or type one in directly. Click Apply to save. This sets the mode to preferred automatically. For entries with @db checked, the numeric sequence becomes editable too — the glyphs you pick there are only used when the @db suffix is present in the input.

Using @db in renderer input

On any renderer page (e.g. text-to-sitelen), you can suffix a name with @db to control how the numeric part is rendered. The suffix goes on the last word of the name.

  • Sema Pan Neninimen — DB is consulted for Sema Pan (glyph cartouche from database), Neninimen renders as a nanpa-linja-n numeric cartouche.
  • Sema Pan Neninimen@db — Same as above, but Neninimen is also overridden with the glyph cartouche stored in the database (only works if the @db checkbox is checked for this entry).
  • Nemen@db — A purely numeric name: without @db it renders as a number; with @db it renders as a glyph cartouche from the database.
  • If the name is not in the database, or the @db checkbox is not checked, the @db suffix is silently stripped and normal rendering applies.

Tips

  • Random cartouches are computed once per page load — the same name always looks the same everywhere on that page, even across multiple renders.
  • Use ignore mode to suppress a name from the database without deleting it — useful for names where you want the renderer's default behaviour.
  • Use Export / Import to back up your database or share entries between sites.
